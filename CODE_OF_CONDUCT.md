# Code of Conduct

## Our Standards

Steinz Labs is built and maintained by a small team. We expect everyone who contributes to or interacts with this project — collaborators, contractors, security researchers, and users — to behave professionally and treat others with respect.

Specifically:

- **Be direct, be kind.** Critique the work, not the person. Phrase feedback so the recipient can act on it.
- **No harassment.** No personal attacks, slurs, sexualized comments, or sustained disruption of discussion. This applies in commits, PRs, issues, and any channel that bears the project's name.
- **No discrimination.** We will not tolerate behavior that targets people based on race, gender identity, sexuality, religion, nationality, age, or disability.
- **No retaliation against security researchers.** If someone reports a vulnerability in good faith per [SECURITY.md](./SECURITY.md), we engage with the report; we do not retaliate.
- **Respect privacy.** Don't share another person's private information without their explicit permission, even if it's technically public.

## Scope

This Code of Conduct applies in any space where the Steinz Labs project is represented — the GitHub repository, the Telegram bot, the Discord server, conference talks, social media accounts under the project's name, and any private 1:1 communication that arises out of project work.

## Enforcement

Reports go to **conduct@nakalabs.xyz**. The maintainers will review every report, ask for additional context if needed, and respond with the action taken. Possible actions include a private warning, a temporary block from the project, or — for serious or repeated violations — a permanent ban.

Reports are kept confidential. Reporters will not be exposed to the person being reported without explicit consent.

The maintainers themselves are not exempt. If a maintainer violates this Code, report it the same way; the remaining maintainers will review.
