## 1.7 BUILD CONFIGURATION ISSUES

| Issue | Location | Risk |
|-------|----------|------|
| typescript.ignoreBuildErrors: true | next.config.js:9 | TypeScript errors silently swallowed in production builds |
| eslint.ignoreDuringBuilds: true | next.config.js:6 | ESLint violations never block deployment |
| reactStrictMode: false | next.config.js:14 | React double-render protection disabled |
| In-memory cache only (Map) | lib/api/cache-manager.ts | Cache lost on every cold start — all Vercel/serverless instances have separate caches. TTLs are meaningless at scale. |
| No CSP header | middleware.ts | Content-Security-Policy header missing from security headers |
| JWT not cryptographically verified | middleware.ts | Middleware only decodes base64 payload and checks exp/role fields — signature not verified |
